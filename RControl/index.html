Que es Remedy Control?

Remedy Control es una herramienta para integrar sistemas (gestores) con BMC Remedy Cloud, proporcionando los recursos necesarios para establecer un medio de comunicación entre el gestor y Remedy Cloud, mediante el uso de las APIs nativas de Remedy Cloud.

Actualmente cuenta con dos medios de comunicación los cuales se describen a continuación.

Componentes Remotos

En este tipo de implementación el gestor utiliza Shells para la invocación de los servicios de Remedy Cloud, los shells se conectan a Remedy Control vía SSH o RSH, el resultado es una cadena con la información recibida de Remedy Cloud.

Servicios WEB

En este tipo de implementación el gestor utiliza un URL para la invocación de los servicios de Remedy Cloud, por lo que solo es necesario poder invocar el servicio web desde el gestor, el resultado se regresa en un formato XML el cual contiene toda la información recibida de Remedy Cloud.

También para esta implementación se cuentan con un cliente desarrollado en Perl para el consumo del Remedy Control , el cual brinda un resultado idéntico a la cadena de los componentes remotos.

Requisitos

A continuación se describen los requisitos mínimos necesarios para la implementación de Remedy Control:

  • 1.- Solicitud de proyecto firmada (indicar el numero de formato de @doc)
  • 2.- Configuración de red con acceso a los servidores de Remedy Control:
    Servidor IP
    producción http://remedycontrol.oymcloud.americamovil.com:8080
    QA http://remedycontrolq.oymcloud.americamovil.com:8080
    desarrollo http://rc01d.oymcloud.americamovil.com:8080
  • 3.- Solicitud de un usuario de Remedy Cloud con permisos en los formularios para consulta y actualización según sea el caso.

    Nota: Las cuentas genéricas para las integraciones deben tramitarse a través de una solicitud ADM en Remedy Cloud por el jefe responsable de la integración. Las cuentas genéricas en Remedy Cloud deben tener licencia fija para soportar multisesiones.

  • Solicitud al área de O&M Departamento de Sistemas de Gestion de Procesos la definición de los formularios a los que se tendrá acceso (ID de columnas, tipos de columnas, valores de columnas, restricciones de columnas, restricciones por registro).

Componentes Remotos

Hardware

  • 1 MB Disco Duro Libre para componentes

  • 1 GB Disco Duro Libres para LOGs de Errores.

  • 1 GB Memoria.

  • Tarjeta de Red.

Software

  • Sistema operativo (Linux, Unix, Solaris 8.0, Microsoft Windows) o superiores

  • J2SDK o JRE 1.7.0_09-icedtea o superior.

Configuración

Los componentes se pueden conectar vía SSH o RSH con el servidor de Remedy Control para lo cual se puede definir 2 tipos de configuración dependiendo de las necesidades:

1.- SSH Requiere generar las llaves de conexión y configurarla en el equipo cliente y en Remedy Control.

2.-RSH Requiere configurar el equipo Remedy control para aceptar las conexiones del equipo cliente.

3.- Usuario de sistema operativo en el equipo cliente con permisos de ejecución de los componentes y permisos para la ejecución de SSH o RSH hacia el servidor de Remedy Control.

4.- Usuario de sistema operativo en Remedy Control con permisos de ejecución de los componentes de Remedy Control.

5.- En el caso de usar la funcionalidad de archivos adjuntos el usuario de sistema operativo deberá tener permisos de FTP para montar los archivos en la carpeta “/usr/archivos” antes de ejecutar la instrucción de insert.

Instalación

Se describen los pasos a seguir para la instalación de los componentes que comprenden Remedy Control y que deberán ser instalados en el equipo donde serán ejecutados.


Archivos de instalación

Los archivos pueden ser descargados desde la dirección:
Servidor link
producción http://remedycontrol.oymcloud.americamovil.com:8080/Remedy
QA http://remedycontrolq.oymcloud.americamovil.com:8080/ Remedy
desarrollo http://rc01d.oymcloud.americamovil.com:8080/Remedy
Para poder ingresar a la páginas se deberá insertar el usuario y la contraseña, ir al menú de sección de soporte y elegir la opción de componentes de sistema donde se encontraran los siguientes archivos disponibles:

  • rmt001.sh Script para consulta de sitios

  • rmt002.sh Script para crear registros

  • rmt003.sh Script para consulta de registros

  • rmt004.sh Script para actualización de registros

  • util.jar Librerías de componentes

Configuración

    1.- Debe estar instalado el JDK descrito en la sección de requerimientos y en las variables de ambiente debe estar definido el PATH con la ruta de instalación por ejemplo: PATH=<ruta jdk>:<ruta jdk>/bin:<ruta jdk>/lib, para comprobar que la configuración del JDK es la correcta desde la ruta de instalación de los archivos de Remedy Control ejecutar el siguiente comando: java –version, el cual debe entregar como respuesta el número de versión de JDK que se encuentra configurado en el sistema operativo.

    2.- Depositar los archivos en una carpeta definida por el usuario por ejemplo: (/user/remedycontrol).

    3.- En caso de requerirlo asignar los permisos de ejecución de los archivos con extensión .sh.

    4.- Modificar los archivos con extensión .sh para asignar las variables de ambiente y parámetros necesarios para su ejecución como se describe a continuación:

      Dentro de los archivos después del encabezado se encuentra una sección que dice “Variables de Configuración” que son las únicas variables que requieren configuración.

      Modificar la variable vsConexion=SSH o vsConexion=RSH para indicar qué tipo de conexión se establecerá con el servidor de Remedy Control.

      Modificar la variable vsEquipo=100.127.4.180 para definir el equipo al que se conectara (Producción, QA o Desarrollo)

      Modificar la variable vsUsuario=<usuario> con el usuario de sistema operativo configurado tanto en el equipo cliente como en el servidor de Remedy Control con permisos para ejecutar las librerías.

      Modificar la variable DIR_HOME=<ruta>/remedycontrol para definir la ruta donde se localizan los archivos de instalación.

Parámetros

No.

Descripción

Condiciones

Clave del sistema o gestor

Esta clave es definida por gestor por ejemplo: TEMIP, FMX.

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

  • sh rmt001.sh "SISTEMA" "sitio"

Respuestas

La ejecución del comando entrega una cadena con el resultado de ejecución el cual puede ser:

  • Cadena con los valores recuperados <site> <nombre> <región> <tecnología> <tipo> <belongsto> <responsablegroup> <estatus>, XE: SITE:DF0001 NOMBRE:ALAMEDA SUR REGION:Region 9 TECNOLOGIA:GSM TIPO:BTS / NODO TX BELONGSTO:CARTR1 RESPONSABLEGROUP:CARRASCO ESTATUS:4 Operando.

  • En caso de no existir el registro del sitio la cadena devuelta es: NO EXISTE REGISTRO RSL: no rows selected

  • En caso de existir un error la cadena puede venir vacía o un mensaje de error de conexión por el SSH..

Creación de Registros (rmt002.sh)

Script para la creación de registros en Remedy Cloud, el cual permite generar el registro en el formulario necesario y también permite validar si el registro ya existe antes de su creación.

Parámetros

No.

Descripción

Condiciones

1

Clave del sistema o gestor

Esta clave es definida por gestor por ejemplo: TEMIP, FMX.

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

2

Formulario donde se creara el registro

Debe ser un formulario valido en Remedy Cloud por ejemplo: HPD:Help Desk , TMS:TASK

3

Columnas para la creación del registro

Las Columnas deben tener el siguiente formato:

· La columna y el valor entre comillas simples y separados por el símbolo de igual(=)

· Si se desea adjuntar un archivo se debe colocar el nombre del archivo entre las etiqueta FILE XE: <FILE>archivo.doc</FILE>, los archivos ya deben estar depositados en la carpeta de carga de archivos en el servidor de Remedy Control.

Por ejemplo:

"'Columna1'='Valor1' 'Columna2'='Valor2' 'Columna..N'='Valor..N'"

“’Columna1’=’Valor1’ ‘Columna2’=’<FILE>archivo.doc</FILE>’”

4

Condiciones para validar si existe el registro antes de ingresarlo.

Si no se desea validar un registro previo se puede omitir este parámetro.

Las Condiciones deben tener el siguiente formato:

· La columna y el valor entre comillas simples(') y separados por el símbolo de igual(=), diferente (<>) o (¡=), like (%=),

· Si se desea usar like en una condición se puede usar también % en el valor

· Si se desea múltiples valores usar (valor1,valor2,valor..N)

· En el caso de múltiples consultas se deben separar por pipes dobles (||)

Por ejemplo:

"'Columna1'='Valor1' 'Columna2'='%Valor2%' 'Columna..N'='Valor..N' || 'Columna1'='Valor1' 'Columna2'='%Valor2%' 'Columna..N'='Valor..N'"

Sintaxis

  • sh rmt002.sh "SISTEMA" "forma" "'columna1'='valor1' 'columna2'='valor2'" "'condicion1'='valor1’ 'condicion2'='valor2'"

Respuestas

La ejecución del comando entrega una cadena con el resultado de ejecución el cual puede ser:

  • Cuando el registro se crea correctamente regresa el número de registro anteponiendo la etiqueta “NVO:”, por ejemplo: NVO:00011111111

  • Cuando el registro existe regresa el número de registro o registros obtenidos anteponiendo la etiqueta “REG [número de validación]” donde el número de validación es en caso de que se defina más de una validación, por ejemplo: REG [1]:000001111

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por la descripción del error, por ejemplo: ERROR: No fue posible crear el registro.

Consideraciones

Si se utiliza el formulario “HPD:Help Desk ”, “TMS:TASK”, y “WOI:WORK ORDER” al enviar la columna correspondiente a la clave del sitio el sistema automáticamente consulta la tabla de “CMDB” para recuperar la información del “Grupo Responsable” “Región” “Site-Belongs” y la agrega a la instrucción insert, en caso de que no se desee la carga automática de estas columnas deberá enviar las columnas con sus valores correspondientes.

En el caso de archivos adjuntos si no existe el archivo, automáticamente por funcionalidad de Remedy Cloud por DEFAULT genera un archivo vacío en el registro.

El archivo deberá estar colocado en la carpeta correspondiente en el servidor de Remedy Control.

Ejercicios

Insertar un Incidente

  • sh rmt002.sh "SISTEMA" "HPD:Help Desk " "'8'='Prueba TT' '536870918'='Mensaje Prueba' '536870912'='10/05/2008 12:30' '536870923'='DF0131'" "'8'='FALLA SPECTRUM' '536870923'='DF0131' '7'='(2,3,1)'"

  • sh rmt002.sh "SISTEMA" "HPD:Help Desk " "'8'='Prueba TT' '536870918'='Mensaje Prueba' '536870912'='10/05/2008 12:30' '536870923'='DF0131' ‘5310102855’=’<FILE>archivo.doc</FILE>’" "'8'='FALLA SPECTRUM' '536870923'='DF0131' '7'='(2,3,1)'"

Insertar una Tarea

  • sh rmt002.sh "SISTEMA" "TMS:TASK" "'536870927'='0000004092' '536870915'='PRUEBA Task (OSCILANDO)' '536870924'='TASK CREADO POR' '536870919'='DF0131'" "'536870927'='0000004092' '536870965'='Finished' '536870915'='PRUEBA (OSCILANDO)'"




Consulta de Registros (rmt003.sh)

Script para la consulta de registros en Remedy , el cual permite obtener información de los registros en cualquier formulario existente en Remedy.

Parámetros

No.

Descripción

Condiciones

1

Clave del sistema o gestor

Esta clave es definida por gestor por ejemplo: TEMIP, FMX.

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

2

Formulario que se desea consultar

Debe ser un formulario valido en Remedy Cloud por ejemplo: HPD:Help Desk , TMS:TASK

3

Columnas que se desea recuperar

Las Columnas deben estar separadas por un espacio y debe ser el numero identificador que le corresponde en Remedy Cloud

Por ejemplo:

"Columna1 Columna2"

4

Condiciones para validar el registro a recuperar

Las Condiciones deben tener el siguiente formato:

· La columna y el valor entre comillas simples(') y separados por el símbolo de igual(=), diferente (<>) o (¡=), like (%=),

· Si se desea usar like en una condición se puede usar también % en el valor

· Si se desea múltiples valores usar (valor1,valor2,valor..N)

Por ejemplo:

"'Columna1'='Valor1' 'Columna2'='%Valor2%' 'Columna..N'='Valor..N’"

5

Parámetros Adicionales

Permite definir el delimitador de resultados y el separador, si se omiten se usaran las de default.

Por ejemplo:

‘DELIMITADOR’=’,’ ‘SEPARADOR’=’||’

Sintaxis

  • sh rmt003.sh "SISTEMA" "forma" "columna1 columna2" "'condicion1'='valor1' 'condicion2'='valor2'"

Respuestas

La ejecución del comando entrega una cadena con el resultado de ejecución el cual puede ser:

  • Cadena con los valores recuperados, cada valor vendrá delimitado por un pipe (|) y las columnas separadas por comas (,), en caso de que sea más de un registro estarán separados por un espacio, por ejemplo: |valor1|,|valor2| |valor1|,|valor2|.

  • Cuando no se encuentra el registro regresa la cadena “NO REG”

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por la descripción del error, por ejemplo: ERROR: Error al realizar la consulta.

Ejercicios

  • Consultar un Incidente

    • sh rmt003.sh "SISTEMA" "HPD:Help Desk " "Columna1 Columna2" "Condiciones de validación"

    • sh rmt003.sh "SISTEMA" "HPD:Help Desk " "1 8" "'8'='%PRUEBA%' '536870923'='DF0131' '7'='(2,3,1)'"

  • Consultar estatus de un Sitio

    • sh rmt003.sh "SISTEMA" "usuario” "Site-EP" "1 7 536870925" "'536870925'='DF0001'"

Actualización de Registros (rmt004.sh)

Script para la actualización de registros en Remedy Cloud, el cual permite modificar información de los registros en cualquier formulario existente en Remedy Cloud.

Parámetros

No.

Descripción

Condiciones

1

Clave del sistema o gestor

Esta clave es definida por gestor por ejemplo: TEMIP, FMX.

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

2

Formulario que se desea consultar

Debe ser un formulario valido en Remedy Cloud por ejemplo: HPD:Help Desk , TMS:TASK

3

Número de registro que se desea actualizar

Es el ID del registro registrado en Remedy Cloud que se desea modificar

4

Columnas a actualizar

Las columnas deben tener el siguiente formato:

· La columna y el valor entre comillas simples(') y separados por el símbolo de igual

· Las columnas deben estar separadas por un espacio

· En caso de realizar más de una actualización usar pipes (||) para separar cada una de las columnas.

· Si se desea adjuntar un archivo se debe colocar el nombre del archivo entre las etiqueta FILE XE: <FILE>archivo.doc</FILE>, los archivos ya deben estar depositados en la carpeta de carga de archivos en el servidor de Remedy Control.

"'Columna1'='Valor1' 'Columna2'='Valor2' 'Columna..N'='Valor..N’"

"'Columna1'='Valor1' 'Columna2'='Valor2' || 'Columna3'='Valor3’"

Sintaxis

  • sh rmt004.sh "SISTEMA" "forma" "ID Registro" "'columna1'='valor1' 'columna2'='valor2'”

Respuestas

La ejecución del comando entrega una cadena con el resultado de ejecución el cual puede ser:

  • Regresa OK en caso de que todos los registros se hayan actualizado correctamente.

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por el número de Incidente y la descripción del error, por ejemplo: ERROR: 00001111111 Error al cerrar el Incidente.

  • http://<ip>/Remedy/servicios/RMDInsert?cSistema=SISTEMA&cForma=Forma&cColumnas='columna1'='valor1' 'columna2'='valor2'&cCondiciones='condicion1'='valor1' 'condicion2'='valor2'

Respuestas

La ejecución del comando entrega una cadena en formato XML con los parámetros enviados y el resultado es regresado en la etiqueta <RESULTADO>RESULTADO</RESULTADO> como se muestra a continuación:


<RMDINSERT>

<SISTEMA>PRUEBA</SISTEMA>

<FORMA>HPD:Help Desk </FORMA>

<COLUMNAS>’Columna1’=’Valor1’ ‘Columna2’=’Valor2’</COLUMNAS>

<CONDICIONES>'Columna1'='Valor1' 'Columna2'='%Valor2%'</CONDICIONES>

<RESULTADO>NVO:000001212</RESULTADO>

</RMDINSERT>


Los posibles resultados pueden ser:

  • Cuando el registro se crea correctamente regresa el número de registro anteponiendo la etiqueta “NVO:”, por ejemplo: NVO:00011111111

  • Cuando el registro existe regresa el número de registro o registros obtenidos anteponiendo la etiqueta “REG [número de validación]” donde el número de validación es en caso de que se defina más de una validación, por ejemplo: REG [1]:000001111

  • Cuando no se encuentra el registro regresa la cadena “NO REG”

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por la descripción del error, por ejemplo: ERROR: No fue posible crear el registro.

Consideraciones

  • Si se utiliza el formulario “Incidente ”, “Tarea”, y “Rutina de Mantenimiento” al enviar la columna correspondiente a la clave del sitio el sistema automáticamente consulta la tabla de “Site-EP” para recuperar la información del “Grupo Responsable” “Región” “Site-Belongs” y la agrega a la instrucción insert, en caso de que no se desee la carga automática de estas columnas deberá enviar las columnas con sus valores correspondientes.

  • En el caso de archivos adjuntos si no existe el archivo, automáticamente por funcionalidad de Remedy Cloud por DEFAULT genera un archivo vacío en el registro.

  • El archivo deberá estar colocado en la carpeta correspondiente en el servidor de Remedy Control.

Consulta de Registros (RMDSelect)

URL para la consulta de registros en Remedy Cloud, el cual permite obtener información de los registros en cualquier formulario existente en Remedy Cloud.

Parámetros

No.

Nombre

Descripción

Condiciones

1

cSistema

Clave del sistema o gestor

Esta clave es definida por gestor por ejemplo: TEMIP, FMX.

El gestor debe estar configurado en Remedy Control ya que mediante este se define que usuario es el que actualiza los registros en Remedy Cloud.

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

2

cForma

Formulario que se desea consultar

Debe ser un formulario valido en Remedy Cloud por ejemplo: HPD:Help Desk , TMS:TASK

3

cColumnas

Columnas que se desea recuperar

Las Columnas deben estar separadas por un espacio y debe ser el numero identificador que le corresponde en Remedy Cloud

Por ejemplo:

"Columna1 Columna2"

4

cCondiciones

Condiciones para validar el registro a recuperar

Las Condiciones deben tener el siguiente formato:

· La columna y el valor entre comillas simples(') y separados por el símbolo de igual(=), diferente (<>) o (¡=), like (%=),

· Si se desea usar like en una condición se puede usar también % en el valor

· Si se desea múltiples valores usar (valor1,valor2,valor..N)

Por ejemplo:

"'Columna1'='Valor1' 'Columna2'='%Valor2%' 'Columna..N'='Valor..N’"

5

cAdicionales

Parámetros adicionales

Permite definir el delimitador de resultados y el separador, si se omiten se usaran las de default.

Por ejemplo:

‘DELIMITADOR’=’,’ ‘SEPARADOR’=’||’

Sintaxis

  • http://<ip>/Remedy/servicios/RMDSelect?cSistema=SISTEMA&cForma=Forma&cColumnas=columan1 columna2&cCondiciones=’condicion1'='valor1' ’condicion2'='valor2'

Respuestas

La ejecución del comando entrega una cadena en formato XML con los parámetros enviados y el resultado es regresado en la etiqueta <RESULTADO>RESULTADO</RESULTADO> como se muestra a continuación:


<RMDSELECT>

<SISTEMA>PRUEBA</SISTEMA>

<FORMA>HPD:Help Desk </FORMA>

<COLUMNAS>’Columna1’=’Valor1’ ‘Columna2’=’Valor2’</COLUMNAS>

<CONDICIONES>'Columna1'='Valor1' 'Columna2'='%Valor2%'</CONDICIONES>

<ADICIONALES>‘DELIMITADOR’=’,’ ‘SEPARADOR’=’||’</ADICIONALES>

<RESULTADO>000001212</RESULTADO>

</RMDSELECT >


Los posibles resultados pueden ser:

  • Cadena con los valores recuperados, cada valor vendrá delimitado por un pipe (|) y las columnas separadas por comas (,), en caso de que sea más de un registro estarán separados por un espacio, por ejemplo: |valor1|,|valor2| |valor1|,|valor2|.

  • Cuando no se encuentra el registro regresa la cadena “NO REG”

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por la descripción del error, por ejemplo: ERROR: Error al realizar la consulta.

Parámetros

No.

Nombre

Descripción

Condiciones

1

cSistema

Clave del sistema o gestor

Existen reglas que aplican por gestor por lo que no se recomienda usar un gestor ya existente a la implementación de otro gestor.

2

cForma

Formulario que se desea consultar

Debe ser un formulario valido en Remedy Cloud por ejemplo: HPD:Help Desk , TMS:TASK

3

cID

Número de registro que se desea actualizar

4

cColumnas

Columnas a actualizar

Las columnas deben tener el siguiente formato:

· La columna y el valor entre comillas simples(') y separados por el símbolo de igual

· Las columnas deben estar separadas por un espacio

· En caso de realizar más de una actualización usar pipes (||) para separar cada una de las columnas.

· Si se desea adjuntar un archivo se debe colocar el nombre del archivo entre las etiqueta FILE XE: <FILE>archivo.doc</FILE>, los archivos ya deben estar depositados en la carpeta de carga de archivos en el servidor de Remedy Control.

"'Columna1'='Valor1' 'Columna2'='Valor2' 'Columna..N'='Valor..N’"

  • http://<ip>/Remedy/servicios/RMDUpdate?cSistema=SISTEMA&cForma=Forma&cID=ID Registro&cColumnas='columna1'='valor1' 'columna2'='valor2'

Respuestas

La ejecución del comando entrega una cadena en formato XML con los parámetros enviados y el resultado es regresado en la etiqueta <RESULTADO>RESULTADO</RESULTADO> como se muestra a continuación:


<RMDUPDATE>

<COLUMNAS>’Columna1’=’Valor1’ ‘Columna2’=’Valor2’</COLUMNAS>

<RESULTADO>OK</RESULTADO>

</RMDUPDATE >


Los posibles resultados pueden ser:

  • Regresa OK en caso de que todos los registros se hayan actualizado correctamente.

  • Cuando se genera un error regresa la etiqueta “ERROR:” seguido por el número de Incidente y la descripción del error, por ejemplo: ERROR: 00001111111 Error al cerrar el Incidente.

Configuración

Al ser un cliente que consume el web services de Remedy Control no se requiere mayor configuración que la integración cuente con un usuario en el sistema

Instalación

Se describen los pasos a seguir para la instalación de los componentes que comprenden Remedy Control y que deberán ser instalados en el equipo donde serán ejecutados.


Archivos de instalación

Los programas se pueden descargar en la página de Remedy Control como se indica en parte de Archivos de instalación.

Para poder ingresar a la páginas se deberá insertar el usuario y la contraseña, ir al menú de sección de soporte y elegir la opción de componentes de sistema donde se encontraran los siguientes archivos disponibles:

  • rmd001.pl Script para consulta de sitios

  • rmd002.pl Script para crear registros

  • rmd003.pl Script para consulta de registros

  • rmd004.pl Script para actualización de registros

  • util.jar Librerías de componentes

Configuración

    1.- Deben estar instaladas las librerías mencionadas en la sección de software

    2.- Depositar los archivos en una carpeta definida por el usuario por ejemplo: (/user/remedycontrol).

    3.- En caso de requerirlo asignar los permisos de ejecución de los archivos con extensión .pl.

    4.- Modificar los archivos con extensión .pl para asignar las variables de ambiente y parámetros necesarios para su ejecución como se describe a continuación:

      De los archivos se encuentra una variable llamada “PRIMERO” la cual contiene la url destino. Esta variable debe de ser modificada con la dirección DNS correspondiente (desarrollo, QA o producción)

*Nota: El uso de estos Scripts es identico al de los componentes remotos

Errores

Los componentes pueden generar una serie de errores los cuales pueden ser manejados internamente, es importante leer muy bien los mensajes de error para identificar los problemas comunes cuando es por usuario de sistema operativo, usuario de Remedy, permisos, columnas inválidas o restricciones del formulario.
